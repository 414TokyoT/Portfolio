Building the Optimal Portfolio: My Modern Portfolio Theory Project in Python

A practical dive into risk, return, and optimization

Setting the Stage

I picked five stocks across sectors to keep things focused: NVDA, JNJ, KO, XOM, and JPM. The goal was to use Modern Portfolio Theory to build two portfolios: one that maximizes risk adjusted return and one that minimizes overall volatility.

How I Approached It

  • Collected five years of adjusted close prices.
  • Converted prices to returns and built an annualized covariance matrix.
  • Ran optimization with PyPortfolioOpt to get Max Sharpe and Min Volatility portfolios.
  • Plotted allocations and summarized expected return, volatility, and Sharpe.

A Quick Note on MPT

MPT is about how assets work together. Two risky assets can combine into a less risky portfolio if they are not perfectly correlated. The optimizer uses expected returns, volatility, and covariance to find the most efficient mix for a given objective.

Results

Max Sharpe Portfolio

Expected Return ~ 86.6%, Volatility ~ 53.9%, Sharpe ~ 1.55. The optimizer concentrated almost entirely in NVDA because its historical return was very high relative to its volatility.

Min Volatility Portfolio

Expected Return ~ 6.78%, Volatility ~ 17.7%, Sharpe ~ 0.21. This leaned heavily into JNJ and KO which are stable and low volatility to reduce overall risk.

My Takeaways

The Max Sharpe solution is mathematically efficient but concentrated. The Min Vol solution is stable but sacrifices upside. Real portfolios add constraints to balance the two such as caps on single name weights, sector limits, or additional asset classes like bonds and gold.

Conclusion

This project made the risk return tradeoff feel concrete. MPT is a great map, but judgment and constraints make it investable. Next I plan to re run the model with weight caps and a broader asset set to see how the efficient set changes.
